# Project Summary

AegisFlow AI is an autonomous data reliability and incident-management platform designed to demonstrate the intersection of artificial intelligence and modern data engineering. It continuously presents the health, freshness, ownership, and success rate of production-style data pipelines. When a failure is simulated, the platform classifies the incident, determines a likely root cause, assigns a confidence score, proposes a safe remediation plan, and waits for explicit human approval before changing pipeline state. Once approved, the remediation workflow restores the pipeline, resolves the incident, updates reliability metrics, and writes an auditable event explaining who authorized the action and what was performed.

The application uses a Next.js and TypeScript frontend, a FastAPI backend, SQLAlchemy persistence, PostgreSQL compatibility, SQLite fallback, Docker Compose, REST/OpenAPI interfaces, automated testing, and GitHub Actions. It is designed as a deployable portfolio system rather than a static dashboard: the frontend and backend can be independently hosted, the persistence layer can be changed through an environment variable, and the incident engine can later receive real telemetry from Kafka, Airflow, dbt, Spark, Great Expectations, or OpenLineage.
